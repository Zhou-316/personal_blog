# 可选 / 拓展功能记录

> 当前阶段先不实现以下功能。项目优先完成方案 2 的基础验收要求：用户、博客、评论、点赞四大模块，以及可部署的网站。基础功能完成并稳定后，再从本文件中挑选 1-2 个进阶功能实现。

## 当前不做的原则

1. 不在基础功能完成前引入复杂 AI、推荐、MCP、游戏等模块。
2. 不为了功能数量牺牲基础链路：注册登录、发文、看文、评论、点赞、部署必须先跑通。
3. 拓展功能后续优先选择“容易展示、容易解释、能体现技术理解”的方向。

## 后续优先推荐的 1-2 个进阶功能

基础功能完成后，优先考虑以下方向：

1. **文章搜索**：容易展示，和博客主题强相关，验收时好解释。
2. **数据统计 / 热度排行榜**：实现成本适中，能体现数据库设计和后端聚合查询。
3. **AI 文章总结**：展示效果明显，但依赖模型 API 或本地模型，放在基础功能之后更稳妥。
4. **MCP Server**：契合文档方向，但复杂度较高，建议在基础功能非常稳定后再做。
